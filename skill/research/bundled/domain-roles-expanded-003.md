## 1. PRODUCT MANAGEMENT

### 1.1 Core Concepts & Keywords

**Product Strategy & Planning**
- Product roadmap development, roadmap prioritization, product vision, strategic planning
- OKRs (Objectives and Key Results), KPIs, performance metrics, success metrics
- Go-to-market (GTM) strategy, product launch management, product lifecycle management
- Product-market fit, product discovery, requirements gathering, product discovery

**Execution & Methodology**
- Agile, Scrum, Kanban, sprint planning, backlog management, backlog prioritization
- User story writing, acceptance criteria, JIRA, Azure DevOps, Confluence, Notion
- Cross-functional team leadership, stakeholder management, stakeholder alignment
- Prioritization frameworks: RICE (Reach × Impact × Confidence ÷ Effort), MoSCoW (Must/Should/Could/Won't)

**Analytics & Validation**
- A/B testing, experimentation, Optimizely, Google Optimize
- Product analytics, Mixpanel, Amplitude, Google Analytics, dashboards
- Market research, competitive analysis, voice of the customer (VoC)
- Customer feedback loops, user testing, data-driven decision making

**Business Impact Metrics**
- Revenue growth, ARR/MRR, churn reduction, retention rate, NPS
- Feature adoption, engagement (DAU/MAU), conversion rate, time-to-market
- Monetization, pricing strategy, customer acquisition cost (CAC), LTV

### 1.2 Must-Have Keywords (ATS-Optimized)

| Tier | Keywords |
|------|----------|
| **Essential** | Product roadmap, OKRs, KPIs, Agile, Scrum, JIRA, backlog, stakeholder management, cross-functional, go-to-market |
| **High-Value** | User stories, A/B testing, market research, competitive analysis, product lifecycle, prioritization |
| **Differentiators** | RICE, MoSCoW, product discovery, product-market fit, customer journey mapping |

### 1.3 Bullet Structure

**Formula**: `[Action verb] + [Task/initiative] + [Metric] + [Business impact]`

**Strong examples:**
- Led cross-functional redesign of checkout flow, boosting conversion by 18%, resulting in $1.2M additional revenue
- Defined and tracked OKRs across 3 product lines; achieved 42% YoY user growth through improved onboarding flows
- Prioritized backlog using RICE framework; reduced time-to-market by 40% while improving feature adoption by 25%
- Launched GTM strategy for new SaaS product; drove $5M ARR in first 12 months

**Metrics to emphasize**: Revenue ($, %), engagement (DAU/MAU %), retention (%, churn reduction), time-to-market (%), conversion (%), NPS

### 1.4 Demonstrating Level

| Level | Signals |
|------|---------|
| **PM** | Owns single product/feature; works with 1–2 teams; executes roadmap |
| **Senior PM** | Owns product area; influences 3+ teams; defines OKRs; leads GTM |
| **Principal/Lead PM** | Owns portfolio; sets strategy; executive stakeholder alignment; P&L awareness |

### 1.5 Common Interview Topics

- **Prioritization**: How do you prioritize features? (RICE, MoSCoW, business impact)
- **Roadmap**: How do you build and communicate a product roadmap?
- **Stakeholder conflict**: How do you handle disagreements with engineering/design?
- **Metrics**: How do you measure product success?
- **Product design**: Design [X] for [audience] (e.g., design a parking app)
- **Behavioral**: STAR format for past failures, influence without authority, trade-off decisions

## 2. CYBERSECURITY

### 2.1 Core Concepts & Keywords

**Compliance & Governance**
- SOC 2 (Type I/II), ISO 27001, ISO 27002, NIST 800-53, NIST 800-171
- FedRAMP, HITRUST, PCI-DSS, HIPAA, SOX (Sarbanes-Oxley), GDPR
- GRC (Governance, Risk, Compliance), risk assessment, audit coordination
- Control framework, policy development, compliance auditing

**Architecture & Strategy**
- Zero trust, zero-trust architecture, defense in depth
- Cloud security (AWS, Azure, GCP), IAM (Identity and Access Management)
- RBAC (Role-Based Access Control), least privilege, MFA/2FA
- Container security (Docker, Kubernetes), Kubernetes security

**Operations & Detection**
- SIEM (Splunk, QRadar, Azure Sentinel), SOC operations
- EDR (CrowdStrike, Carbon Black, SentinelOne), XDR
- Threat detection, threat hunting, vulnerability management
- Incident response, DFIR (Digital Forensics & Incident Response)
- MITRE ATT&CK, Cyber Kill Chain, OWASP

**DevSecOps & Automation**
- Shift-left security, secure SDLC, SAST, DAST
- CI/CD security, Jenkins, GitHub Actions, GitLab CI
- Terraform, Checkov, Snyk, Veracode, Aqua, Anchore
- Infrastructure-as-Code (IaC) security

### 2.2 Must-Have Keywords (ATS-Optimized)

| Tier | Keywords |
|------|----------|
| **Essential** | SOC 2, ISO 27001, NIST, incident response, vulnerability management, IAM, RBAC |
| **High-Value** | DevSecOps, SIEM, zero trust, threat modeling, compliance (PCI-DSS, HIPAA, SOX) |
| **Differentiators** | MITRE ATT&CK, container security, FedRAMP, GRC |

### 2.3 Positioning Infra/DevOps for Security Roles

**Bridge skills to emphasize:**
- Security tool integration in CI/CD pipelines
- Container/Kubernetes security (Docker Bench, KubeBench, KubeHunter)
- IaC with security baselines (Terraform, CloudFormation, Ansible)
- IAM policies, encryption, security configurations in cloud
- Vulnerability scanning automation (Snyk, Aqua, Veracode)
- Compliance automation (SOC 2, FedRAMP evidence collection)

**Bullet examples for transition:**
- Integrated Snyk and Checkov into CI/CD pipeline; reduced critical vulnerabilities by 60% pre-production
- Implemented Kubernetes security baseline using KubeBench; achieved zero critical findings in 3 audit cycles
- Automated SOC 2 evidence collection via Terraform; cut audit prep time by 40%

### 2.4 Bullet Structure

**Formula**: `[Action verb] + [Security initiative] + [Tool/framework] + [Metric/outcome]`

**Strong examples:**
- Led SOC 2 Type II certification; achieved zero deficiencies across 5 trust service criteria
- Implemented zero-trust architecture for 200+ microservices; reduced blast radius of credential compromise by 80%
- Established incident response playbooks aligned to MITRE ATT&CK; reduced MTTR from 4 hours to 45 minutes
- Automated vulnerability scanning across 50+ container images; remediated 90% of critical CVEs within 7 days

### 2.5 Demonstrating Level

| Level | Signals |
|------|---------|
| **Analyst** | SOC operations, alert triage, SIEM dashboards, playbook execution |
| **Engineer** | Tool implementation, automation, vulnerability remediation, DevSecOps |
| **Architect** | Zero trust design, threat modeling, compliance program design |
| **Manager/Lead** | Team leadership, audit ownership, vendor evaluation, board reporting |

### 2.6 Common Interview Topics

- **Incident response**: Walk through a security incident you handled
- **Threat modeling**: How do you threat model a new system?
- **SOC 2/Compliance**: Describe your experience with SOC 2 or ISO 27001
- **Zero trust**: How would you implement zero trust in an existing environment?
- **SIEM**: How do you build detection rules? (correlation, tuning, false positives)
- **DevSecOps**: How do you integrate security into CI/CD?

## 3. SOFTWARE ENGINEERING

### 3.1 Core Concepts & Keywords

**Architecture**
- System design, distributed systems, microservices architecture
- Event-driven architecture, message queues (Kafka, RabbitMQ)
- Cloud-native (AWS, GCP, Azure), Kubernetes, Docker
- Scalability, high availability, fault tolerance, load balancing
- Database design, sharding, replication, caching (Redis, Memcached)

**API & Protocols**
- REST, GraphQL, gRPC, API design, API gateway
- Protocol Buffers, OpenAPI/Swagger
- Service-to-service communication, low-latency systems

**Languages & Stack**
- Python, TypeScript, Java, Go, Rust
- Backend, frontend, full-stack

**Quality & Process**
- CI/CD, code review, testing (unit, integration, e2e)
- SRE, SLOs, SLIs, observability, monitoring
- Technical debt, refactoring, performance optimization

### 3.2 Must-Have Keywords (ATS-Optimized)

| Tier | Keywords |
|------|----------|
| **Essential** | System design, microservices, distributed systems, API design, CI/CD, scalability |
| **High-Value** | REST, GraphQL, Kubernetes, cloud (AWS/GCP/Azure), performance, code review |
| **Differentiators** | gRPC, event-driven, SRE, platform engineering |

### 3.3 Bullet Structure

**Formula**: `[Action verb] + [Technical initiative] + [Scale/constraint] + [Metric]`

**Strong examples:**
- Architected event-driven microservices platform processing 10M events/day; reduced latency by 60%
- Designed GraphQL API layer for 15+ client applications; eliminated over-fetching, cutting payload size by 40%
- Led migration from monolith to Kubernetes-based microservices; improved deployment frequency from weekly to 50/day
- Implemented distributed caching layer (Redis); reduced database load by 70%, p99 latency from 800ms to 120ms

### 3.4 Staff/Principal Level Expectations

| Expectation | How to Show |
|-------------|-------------|
| **Cross-org influence** | "Drove adoption of X across 5 teams"; "Established standards used org-wide" |
| **Technical strategy** | "Defined architecture strategy for Y"; "Evaluated build vs buy for Z" |
| **Mentorship** | "Mentored 3 engineers to promotion"; "Led architecture review board" |
| **Business impact** | Tie technical decisions to revenue, cost, or reliability (e.g., "$2M cost reduction") |
| **Scope** | Multi-team, platform, or org-level initiatives |

### 3.5 Common Interview Topics

- **System design**: Design Twitter, Netflix, Uber, notification system, rate limiter, etc.
- **Clarifying questions**: Users, scale, constraints before proposing solution
- **Tradeoffs**: Why REST vs GraphQL? Why Kafka vs RabbitMQ?
- **Scalability**: Sharding, caching, load balancing, database design
- **Failure modes**: What happens when X fails? How do you handle partial failures?
- **Coding**: LeetCode-style (often lighter at Staff+) or domain-specific

## Appendix A: Cross-Role Bullet Framework (CAR)

**Challenge**: Context or problem  
**Action**: What you did (verb + method/tool)  
**Result**: Quantified outcome + business impact  

Use for all roles; adjust vocabulary per domain.
